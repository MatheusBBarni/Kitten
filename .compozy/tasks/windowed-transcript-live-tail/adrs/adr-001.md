# ADR-001: Ship a flagged bounded live transcript projection

## Status

Accepted

## Date

2026-07-15

## Context

Kitten renders every retained turn in the focused session even though long-running coding-agent sessions need only a small working view. This causes presentation work to grow with transcript length while the product must retain every live-run turn for inspection. The runtime intentionally does not persist transcript content to disk, so a collapsed-history UI must not imply cross-restart recovery. Tool updates may target older turns, and approval and clarification state live outside transcript turns.

## Decision

Ship a minimal, default-off bounded transcript presentation feature. The authoritative live-run transcript remains complete in core memory. A pure projection presents a protected tail containing active streaming work, pending or running tools, and recent conversational turns; older content becomes an identity-stable collapsed-range marker that users can explicitly expand. Projection state is ephemeral and independent per session. The feature must state its live-run scope honestly, deterministically re-project older tool updates, preserve a manually detached scroll anchor on history expansion, and emit only opt-in, content-free performance counters.

## Alternatives Considered

### Alternative 1: Continue rendering the full transcript

- **Description**: Keep the current direct full-transcript rendering path and optimize individual rows.
- **Pros**: No presentation-state or history-marker behavior to introduce.
- **Cons**: Rendering and reconciliation continue to scale with session length; long-session reliability remains unaddressed.
- **Why rejected**: It fails the primary user outcome: a responsive, stable cockpit for extended runs.

### Alternative 2: Add generic list virtualization first

- **Description**: Adopt a general-purpose virtualization layer while retaining one continuously rendered transcript model.
- **Pros**: Familiar conceptual model and potential broader reuse.
- **Cons**: Adds substantial UI and measurement complexity without first proving a bounded projection solves the problem; does not by itself define protected live content or history semantics.
- **Why rejected**: It is a larger, less reversible first step than the required product learning loop.

### Alternative 3: Add transcript persistence and a full history platform

- **Description**: Persist transcript content and build configurable history, search, and restoration around it.
- **Pros**: Could support cross-restart history and richer navigation.
- **Cons**: Changes Kitten's privacy contract, storage schema, and scope materially.
- **Why rejected**: Cross-restart persistence is not needed to prove the live-session responsiveness hypothesis and is explicitly out of V1.

## Consequences

### Positive

- Bounds the active presentation tree while retaining the complete live-run transcript.
- Protects current agent work and tool activity from being hidden by history compaction.
- Keeps the policy core-owned and per-session display state reversible.
- Provides a rapid disable path while real-session evidence is gathered.

### Negative

- The history marker introduces new anchor, expansion, and old-tool-update behavior to test.
- A default-off flag delays universal availability until the behavior is proven in use.
- The product must be precise that expansion recovers live in-memory history, not disk-persisted transcript history.

### Risks

- **Hidden or stale work**: protect live/tool rows and deterministically re-project all tool updates.
- **Scroll jumps**: test detached-scroll prepends and preserve a stable visual anchor.
- **Misleading recoverability expectations**: communicate the live-run-only lifetime and retain the existing no-transcript-persistence contract.
- **Unproven operational behavior**: use a simple flag and content-free measurements before widening rollout.

## Implementation Notes

Keep the semantic transcript authoritative in core; isolate window selection from view rendering; keep per-session expansion and scroll state ephemeral; defer persistence, search, configurable policies, and generalized virtualization.
