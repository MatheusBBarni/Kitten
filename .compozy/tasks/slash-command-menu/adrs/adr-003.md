# ADR-003: Surface agent commands as a `config_options`-style domain slice

## Status

Accepted

## Date

2026-07-10

## Context

The focused agent advertises its slash commands over ACP as an `available_commands_update` session notification, but `acpTranslate.ts` currently maps that variant to `null` (dropped, "not surfaced in V1").
The slash menu needs the focused agent's live command list as reactive state that re-renders when the agent adds, removes, or changes commands mid-session.

Kitten already surfaces two analogous per-session lists through the same pipeline: `plan` (latest-wins) and `configOptions` (wholesale-replace of an advertised set translated from ACP). `config_options` is the closer structural match, since it translates ACP fields into a protocol-free domain shape and replaces the whole set on each update - exactly the semantics of an advertised command list.

The ACP `AvailableCommand` carries `name`, `description`, an optional `input` (whose `UnstructuredCommandInput` has a required `hint`), and `_meta`. The `acpTranslate` completeness test (`FORBIDDEN_ACP_KEYS`) forbids `_meta` from ever appearing in a translated domain event.

## Decision

Add a `commands` slice that mirrors the existing `configOptions` flow end to end:

- **Domain type** (protocol-free, in `src/core/types.ts`): `interface AvailableCommand { name: string; description: string; hint?: string }`. Flatten ACP's `input.hint` into `hint`; drop `input` wrapper and `_meta`.
- **Domain event**: add `| { kind: "commands"; commands: AvailableCommand[] }` to `DomainSessionEvent`. The `assertNever` guard in the reducer forces the matching case to exist.
- **Translation** (`acpTranslate.ts`): replace the `available_commands_update` no-op with a `case` that maps `update.availableCommands` to the domain shape via a `translateCommand` helper, mirroring `translateConfigOptions`.
- **Reducer** (`sessionReducer.ts`): a `case "commands"` that wholesale-replaces `state.commands`, plus `commands: []` in `createSessionState`.
- **State field**: `commands: AvailableCommand[]` on `SessionState`.
- **Selector** (`selectors.ts`): `selectSessionCommands(sessionId)` mirroring `selectSessionPlan`, memoized at the call site.

No ACP type crosses `src/agent/`. The menu reads `selectSessionCommands(focusedSessionId)`.

## Alternatives Considered

### Alternative 1: Fetch commands on demand from the connection

- **Description**: Have the menu ask the `AgentConnection` for the current command list when it opens, instead of storing them.
- **Pros**: No new reducer state.
- **Cons**: ACP pushes commands (server notification), it is not a request/response; the UI would reach past `ControllerActions` into a connection, violating the layering rule; and it could not react to mid-session changes.
- **Why rejected**: Contradicts the store-is-the-single-source and UI-only-through-selectors boundaries.

### Alternative 2: Store the raw ACP `AvailableCommand[]` directly

- **Description**: Keep the ACP objects and let the menu read `input.hint` and ignore `_meta`.
- **Pros**: No translation code.
- **Cons**: Leaks an ACP wire type into core/store/UI, breaking the anti-corruption boundary and the `FORBIDDEN_ACP_KEYS` test (`_meta`).
- **Why rejected**: Directly violates the ACP-only-in-`src/agent/` rule.

## Consequences

### Positive

- Reuses a proven, tested pipeline; the command list is reactive and mid-session-safe by construction.
- The compiler (via `assertNever`) and the completeness test enforce correctness.
- Menu code depends only on a selector and a protocol-free type.

### Negative

- Adds one more per-session field and its initializer, translation, reducer case, and selector (five small edits across the vertical slice).

### Risks

- The `acpTranslate.test.ts` null-table entry for `available_commands_update` must be removed and replaced with positive-case and reducer round-trip tests, or the suite fails. Mitigation: update the test in the same change.

## Implementation Notes

- `translateCommand(cmd: AcpAvailableCommand): AvailableCommand` returns `{ name, description, hint: cmd.input?.hint }` and nothing else.
- Extend the completeness test to feed `available_commands_update` through `collectKeys` and assert no forbidden keys.
