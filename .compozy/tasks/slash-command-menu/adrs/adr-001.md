# ADR-001: `/` command menu - V1 scope, trigger model, and state ownership

## Status

Accepted

## Date

2026-07-10

## Context

Kitten is a terminal cockpit that runs two AI coding agents (Claude Code and Codex) side by side over the Agent Client Protocol (ACP).
Its capabilities are reachable today only through control/function-key chords (Ctrl+T hand-off, Ctrl+O switch-focus, F1 help), and the focused agent's own advertised slash commands are received over ACP but discarded (`acpTranslate.ts:53` returns `null` for `available_commands_update`).
Discoverability is poor: research on hidden keyboard features (a ~69k-user study found 81% never used Ctrl+F once in a week) implies the marquee Ctrl+T hand-off is invisible to most users.

The proposal is a `/` command menu in the prompt editor that unifies Kitten's own cockpit actions with the focused agent's advertised commands.
The council debate surfaced three decisions that shape V1: the trigger/modality model, how far the ambition should go, and where the new state lives.
A parallel task (Ctrl+S sessions overview) is concurrently editing `keymap.ts`, `appStore.ts`, and `selectors.ts` - the same files this feature touches.

## Decision

Ship a **non-modal, editor-local `/` command palette** in V1 with the following boundaries:

- **Unified, grouped menu.** One searchable surface listing Kitten cockpit actions and the focused agent's ACP-advertised commands, grouped by source (Cockpit / <focused agent>). Cockpit actions rank first so the hand-off is never buried.
- **Trigger = token-begin guard, not "any `/`".** The menu arms only when `/` begins a token (start of input or immediately after whitespace/newline) and the caret sits within that token. The token after `/` filters the list live (`onContentChange`). A no-match result, a second `/` inside the token, or the caret leaving the token **disarms** the menu.
- **Invoke-not-send.** While the menu is armed, `onKeyDown` captures Up/Down/Tab/Enter/Esc (`preventDefault` before the textarea acts). Enter invokes the highlighted item: a Kitten action runs locally (hand-off still opens its preview), an agent command inserts its text into the prompt for manual send. When disarmed, Enter submits the prompt normally. Nothing auto-sends.
- **V1 polish is non-negotiable.** Grouping by source, cockpit-first ranking, keyboard-shortcut hints printed on each cockpit row (teaching surface), and ACP `input.hint` argument hints on agent commands. Not-ready agents' commands are shown-but-disabled with a reason (per the resilience rule).
- **State ownership.** The agent command *data* is domain state: a new `DomainSessionEvent` variant, a reducer case writing a latest-wins `SessionState.commands` field (mirroring `plan`), and a selector. No ACP type escapes `src/agent/`. The *menu* (open/filter/highlight) is ephemeral view state held in local React state inside `PromptEditor`; it never enters the store or `SessionState`.
- **Sequencing.** Land the ACP command vertical-slice as its own PR first (pure-core testable, lowest risk), coordinated with the live Ctrl+S task; keep keymap changes additive.

Defer inline hand-off preview in the dropdown and cross-agent command routing to V2.

## Alternatives Considered

### Alternative 1: Modal menu, `/`-at-start-of-empty-prompt only (pragmatic-engineer)

- **Description**: Reuse the existing modal overlay slot; trigger only when `/` is the first char of an empty prompt; filter inside the menu rather than in the editor.
- **Pros**: Reuses tested key-capture/focus/dismissal infrastructure; eliminates Enter-ambiguity and slash-collision by construction; lowest implementation risk.
- **Cons**: Modality blurs the prompt editor, which is a regression for an autocomplete (the user is still typing the filter); silently overrides the user's locked "filter as you type" intent.
- **Why rejected**: The token-begin guard plus "the render is the signal" (menu armed only when a candidate list is visible) solves the same two bugs without amputating filter-as-you-type. Retained as the **fallback** if the non-modal overlay proves janky in practice.

### Alternative 2: Persistent footer hint for the hand-off, no palette (devils-advocate)

- **Description**: A one-line always-visible affordance teaching Ctrl+T (and other core chords) instead of building a menu.
- **Pros**: Near-free; directly targets the real north-star (first-session hand-off rate); no ACP-boundary risk; ships in days.
- **Cons**: Does not unlock the agent's advertised commands (still 100% unreachable); does not scale to a growing command set; not the differentiated two-agent surface.
- **Why rejected as a replacement**: It solves only the hand-off half of discoverability. **Adopted as a complementary low-cost win** and as a measurement discipline (track first-session hand-off rate regardless of the palette).

### Alternative 3: Literal "any `/` typed" trigger (original locked scope)

- **Description**: Every `/` keystroke anywhere in the prompt opens the menu.
- **Pros**: Simplest mental model; matches the user's first instinct.
- **Cons**: Collides with file paths (`/usr/bin`), URLs (`https://`), regexes, and code on the surface users touch most - a footgun on every slash.
- **Why rejected**: Refined into the token-begin guard, which preserves the "type `/` then filter" feel without ambushing normal typing.

## Consequences

### Positive

- Makes every capability - including the marquee hand-off - discoverable by typing `/`, with the chord taught inline.
- Unlocks the focused agent's advertised commands, which are unreachable today.
- Establishes the two-agent grouped palette as a differentiator no single-agent CLI can match, with a clean V2 path to a true control surface.

### Negative

- Introduces a new non-modal, editor-local overlay concept the codebase does not yet have.
- Key-capture ordering (Enter = accept vs. submit, Esc, dismiss-on-space) adds complexity to the prompt editor, the highest-traffic surface.
- Requires a from-scratch vertical slice through the ACP anti-corruption boundary.

### Risks

- **Trigger edge cases** (paths, URLs, code). Mitigation: token-begin guard + no-match disarm + multi-slash disarm; cover with tests before enabling.
- **Ranking burying important commands** (Claude Code shipped this bug). Mitigation: cockpit-first ranking, deterministic ordering, capped visible rows.
- **Live command state** changing mid-session. Mitigation: treat the command list as reactive selector state, re-rendered on `available_commands_update`.
- **Merge conflict** with the parallel Ctrl+S task. Mitigation: ship the ACP slice as its own PR, keep keymap edits additive, coordinate selector changes.

## Implementation Notes

- ACP slice mirrors the existing `plan` flow end to end: `acpTranslate.ts` case -> `DomainSessionEvent` -> `sessionReducer` case -> `SessionState.commands` -> selector. The `acpTranslate` completeness test (`FORBIDDEN_ACP_KEYS`) must still pass - drop ACP-only keys (`_meta`).
- Menu descriptors (label, group, shortcut, invoke-thunk) are assembled in the view from the `COCKPIT_KEYMAP` table (already the single source of truth for dispatch and help) plus the focused session's command selector. Invoke-thunks route only through `ControllerActions` / the shell `HandoffFlow` - the menu never holds an `AgentConnection`.
- Nav-key capture belongs in the textarea's `onKeyDown` (can `preventDefault` before `PROMPT_KEY_BINDINGS`); token detection in `onContentChange`.
