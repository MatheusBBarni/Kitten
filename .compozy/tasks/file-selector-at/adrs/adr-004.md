# ADR-004: Keep @ Completion Local to the Prompt Token

## Status

Accepted

## Date

2026-07-12

## Context

`PromptEditor` already owns the imperative textarea buffer, cursor offset, slash-token parsing, menu state, and Enter interception. A file selector must let developers continue typing to filter, accept one result without sending the prompt, and dismiss without mutating the draft. The PRD also requires normal `@` text such as email addresses to remain ordinary composition.

## Decision

Extend the editor's local completion state into a discriminated union for slash and file completion. A file token begins only at a whitespace-delimited token boundary (including offset zero) and owns the cursor while its text begins with `@`. The filter is the token text after the prefix.

Render a stateless `FileSelector` above the textarea beside the existing `SlashMenu`. Reuse `MENU_KEYMAP` and `matchMenuCommand`: arrows and Tab move the highlight, Enter replaces only the active `@query` range with a visible reference, and Escape closes the selector without changing the draft. Enter must call `preventDefault()` while a selectable result is armed, so the prompt is not sent.

Format a selected path as `@relative/path` when it contains no whitespace, quote, or backslash; otherwise insert `@` plus a JSON-style quoted and escaped relative path, for example `@"src/My File.ts"`. Reject control-character paths before they reach this layer. The selector only parses unquoted in-progress query tokens; the inserted reference is ordinary visible prompt text after selection.

Escape records a local suppression marker for the active token. While the cursor remains in that token and it still begins at the same offset with `@`, ordinary typing cannot reopen completion. Delete the trigger, move the cursor out of the token, start a different token, or switch focus to clear suppression.

Loading, no-match, and unavailable states remain visible but non-modal. They consume no printable input. Selection changes only prompt text and never writes `referencedFiles`, creates ACP attachment blocks, or sends an agent request.

## Alternatives Considered

### Alternative 1: Global or modal file picker

- **Description**: Open a modal overlay from `@` and move focus away from the textarea.
- **Pros**: Isolates keyboard handling and provides more layout space.
- **Cons**: Interrupts typing-driven filtering and complicates draft focus restoration.
- **Why rejected**: The required interaction is continuous prompt composition, not a separate selection task.

### Alternative 2: Trigger on every @ character

- **Description**: Start completion for `@` anywhere in a token.
- **Pros**: Maximizes opportunities to invoke the feature.
- **Cons**: Hijacks email addresses and embedded ordinary text.
- **Why rejected**: Predictable plain-text composition is more important than aggressive triggering.

### Alternative 3: Store completion state globally

- **Description**: Add query, result, and selection state to `AppStore`.
- **Pros**: State becomes inspectable across components.
- **Cons**: Couples a textarea-local interaction to global mutations and unrelated renders.
- **Why rejected**: The token range belongs to the imperative editor and has no meaning outside it.

## Consequences

### Positive

- Keeps the interaction consistent with existing slash completion.
- Preserves the current draft across acceptance, dismissal, and no-match states.
- Prevents accidental prompt submission when accepting a file.
- Supports whitespace-bearing safe paths without ambiguous prompt references.

### Negative

- `PromptEditor` gains another completion variant, async request guards, and dismissal suppression.
- The file menu shares a compact terminal layout with the slash menu.

### Risks

- **Enter ordering regression**: intercept keys only when the rendered file menu has a selectable row; cover both Enter variants in UI tests.
- **Ambiguous trigger edge cases**: cover beginning-of-prompt, whitespace, email, path, cursor movement, token deletion, and suppression tests.
- **Stale menu state**: invalidate local file state and cache on focused-session changes.

## Implementation Notes

- Keep pure token parsing, path matching, formatting, and range replacement as exported helpers with direct unit tests.
- Keep `FileSelector` presentation-only; it receives rows/status/highlight and owns neither I/O nor mutable state.
- Add `@` to the editor help entries in `keymap.ts`; do not create a second navigation binding table.
