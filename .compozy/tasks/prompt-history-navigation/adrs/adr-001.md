# ADR-001: Scope Prompt Recall to the Active Agent Session

## Status

Accepted

## Date

2026-07-12

## Context

Kitten users need to recover a recently sent prompt, adjust it, and send it again without retyping it. The prompt composer is a multiline terminal textarea where Up and Down also have established cursor-navigation meaning, and its slash menu already uses those keys while open. Prompt content can be sensitive, and Kitten runs multiple agent sessions side by side.

## Decision

Provide a bounded, in-memory prompt history for the current run, isolated to each agent session. In the plain focused composer, Up recalls older submitted prompts and Down returns to newer entries. Down clears the composer only after the user has recalled the newest entry. The history is unavailable while the slash menu is open and must not replace normal vertical cursor movement when the caret can still move inside a multiline or wrapped draft.

## Alternatives Considered

### Alternative 1: Persistent, searchable cross-session history

- **Description**: Save prompts across restarts and expose search or browsing.
- **Pros**: Supports long-term reuse and discovery.
- **Cons**: Introduces retention, sensitive-content, migration, lifecycle, and management concerns.
- **Why rejected**: It broadens a small reuse improvement into a history-management product before demand is validated.

### Alternative 2: Unconditional arrow-key recall

- **Description**: Treat every Up and Down keypress in the focused composer as prompt history navigation.
- **Pros**: Simple mental model for single-line input.
- **Cons**: Breaks expected multiline cursor movement and can accidentally replace or clear a draft.
- **Why rejected**: It makes a common editing action unpredictable.

## Consequences

### Positive

- Delivers quick prompt reuse without persistent prompt storage.
- Keeps prompts private to their originating agent session.
- Preserves the familiar arrow-key behavior of modal menus and multiline editing.

### Negative

- Users cannot retrieve prompts after a restart or from another agent session.
- The interaction has an eligibility boundary that needs clear, predictable behavior.

### Risks

- **Accidental draft loss**: only clear after the newest recalled entry, never from ordinary editing.
- **Cross-session leakage**: keep history owned by and discarded with its session.
- **Key conflicts**: keep slash-menu navigation and vertical cursor movement ahead of recall.
