# ADR-007: Git Branch via Boot plus Turn-Boundary Refresh

## Status

Accepted

## Date

2026-07-09

## Context

The status bar shows the working-directory git branch, a shared segment across both agents.
Git branch has no ACP source; it must be read from each session's `cwd`.
It must never be read on the render path, and it must survive the common case where the user switches branches between tasks.
Each session already carries a fixed `cwd` on `SessionState`.

## Decision

Read the branch with a short git command off the render path, store it, and refresh it at natural boundaries.

- Read `git rev-parse --abbrev-ref HEAD` (or equivalent) in the session `cwd` once at startup and store the result in the store.
- Refresh at natural, low-frequency boundaries: focus switch and turn completion. Never on render.
- Edge cases: detached HEAD -> show a short SHA; non-repo cwd or command failure -> hide the branch segment (hide-when-absent).
- The branch value is exposed through a narrow selector the bar consumes, consistent with the slot contract in ADR-006.

## Alternatives Considered

### Alternative 1: Watch `.git/HEAD`

- **Description**: Install a filesystem watcher per session on `.git/HEAD` (and packed-refs) and update reactively.
- **Pros**: Reflects a mid-task branch switch immediately.
- **Cons**: A watcher per session plus more edge cases (worktrees, submodules, packed refs); more moving parts than the value warrants.
- **Why rejected**: Over-engineered for a signal that changes rarely mid-session; the refresh boundaries already cover the realistic case.

### Alternative 2: Boot-only

- **Description**: Read once at startup and never refresh.
- **Pros**: Simplest.
- **Cons**: Goes stale if the user switches branches during a session, showing the wrong branch until relaunch.
- **Why rejected**: A wrong branch in the bar is worse than a hidden one; cheap refresh boundaries avoid it.

## Consequences

### Positive

- Cheap and simple; correct for the common between-tasks branch change; never touches the render path.
- Degrades cleanly (short SHA for detached HEAD, hidden outside a repo).

### Negative

- A branch switch made mid-turn is reflected only at the next boundary (focus switch or turn completion), a brief staleness window.

### Risks

- **Shell-out latency or failure** blocking a boundary. Mitigation: run asynchronously and treat failure as "hidden"; never block the UI on it.

## Implementation Notes

- Add a small branch-reader utility that runs the git command in a given cwd and returns `{ branch } | { sha } | null`, invoked at boot and at the focus/turn boundaries, writing into the store.
- Store the branch on a store field keyed by session (or a shared field, since both sessions share the launch repo in the common case) and expose it via a narrow selector.
