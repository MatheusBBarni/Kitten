# ADR-003: Headroom derivation - raw usage fact in state, primitive selector, pure formatter

## Status

Accepted

## Date

2026-07-09

## Context

The gauge must render each agent's context headroom in two surfaces (the status strip and the `Ctrl+T` handoff preview) built from the ACP `usage_update` event Kitten currently drops at `src/agent/acpTranslate.ts:57`.
Three forces shape the data flow:

- **Re-render isolation (ADR-004 in the codebase).** Each agent's status chip subscribes only to its own agent; a usage tick for one agent must not repaint the other. Selectors that return fresh objects break `Object.is` short-circuiting and leak renders.
- **Honesty (ADR-001).** Absent data must read as "unknown," never a fabricated `0%`. The carrier of "unknown" must be unambiguous end to end.
- **Layering.** The pure core (`src/core`) holds raw domain facts; presentation math and formatting do not belong in domain state, and there is no existing bar/percentage rendering utility to reuse.

The choice is where the headroom number is computed and how it is formatted for display.

## Decision

Split the concern across three single-purpose seams:

1. **State holds the raw fact.** `SessionState.usage?: { used: number; size: number }`; `undefined` until the agent reports, which is the honest "unknown." The reducer's `case "usage"` mirrors the existing `status` case (replace one field, touch nothing else). `createSessionState` leaves `usage` undefined; a session reset drops stale usage back to unknown for free.
2. **A primitive curried selector computes headroom.** `selectSessionHeadroom(sessionId): Selector<number | null>` returns a rounded remaining-context percentage, or `null` when usage is absent or `size <= 0`. A number/`null` compares by `Object.is`, giving per-agent re-render isolation with no extra equality function, exactly like `selectSessionStatus`.
3. **A pure helper formats the display.** `formatHeadroom(pct: number | null, cells)` returns the percent label plus a fixed-width bar spec, and the "unknown" marker for `null`. It is a pure function, unit-tested like the reducer, and shared by both surfaces so they never disagree.

Cost is not captured (see the PRD Non-Goals and the TechSpec); the translator copies only `used`/`size` and drops `cost`/`_meta`.

## Alternatives Considered

### Alternative 1: Raw-object selector, compute headroom in each component

- **Description**: `selectSessionUsage(id)` returns `{used,size}|undefined`; each UI component computes and formats headroom inline.
- **Pros**: One selector; no separate formatter module.
- **Cons**: Duplicates the headroom math across the status strip and the handoff preview (they can drift); an object-returning selector needs care to stay identity-stable; formatting is not independently unit-testable.
- **Why rejected**: Duplication across the two surfaces and weaker testability outweigh the saved file.

### Alternative 2: Store the computed headroom percentage in state

- **Description**: The reducer computes and stores `headroomPct` on `SessionState`.
- **Pros**: Trivial read path; selector just returns the field.
- **Cons**: Puts derived/presentation data in the pure domain state, breaking the "state holds raw facts" convention; recomputation policy leaks into the reducer; harder to change the display formula later.
- **Why rejected**: Violates the core/presentation boundary for no real gain, since a selector computes the same value cheaply.

## Consequences

### Positive

- Per-agent re-render isolation is preserved by construction (primitive selector).
- "unknown" is unambiguous and honest at every layer (`undefined` fact → `null` selector → marker in the formatter).
- The reducer change is a one-line mirror of `status`; `assertNever` guarantees the case is not forgotten.
- The formatter and selector are pure and unit-testable without rendering.

### Negative

- Introduces one new UI helper module (the formatter/bar) where none existed.
- The headroom formula lives in the selector, so a future change (e.g., accounting for reserved headroom) touches the selector rather than the raw fact - acceptable, and localized.

### Risks

- **Object identity slip**: if a future change makes the selector return a fresh object, isolation breaks. Mitigation: keep the selector primitive (`number | null`); assert identity stability in a selector test.
- **Formula trust**: a naive `(size-used)/size` may not reflect the effective limit under compaction. Mitigation: neutral MVP framing (ADR-002), no verdict; the formula is isolated for later refinement.

## Implementation Notes

- Selector mirrors `selectSessionStatus` (`src/store/selectors.ts:39-42`); consumers memoize the curried call per `sessionId` (as `AgentStatusChip` does).
- The formatter is a new pure module under `src/ui` (e.g. `headroom.ts`), returning the "unknown" marker for `null` and a fixed-width bar for the strip.
- Reuse existing palette tokens for the neutral bar (fill vs `muted` track); no new palette key in the MVP.
