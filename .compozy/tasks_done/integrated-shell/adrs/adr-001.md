# ADR-001: V1 Integrated Shell Is a Real PTY That Feeds the Hand-off

## Status

Accepted

## Date

2026-07-09

## Context

Kitten is a terminal cockpit that runs two AI coding agents side by side and hands a live task between them.
Its differentiator is portable cross-agent context hand-off, not the terminal itself.

The original `IDEA.md` listed a shell as a core feature, but the shipped `kitten-agent-tui` V1 idea **deliberately deferred** it, judging an embedded shell to be "toad's existing strength [that] adds surface without moving the hand-off metric."
This ADR covers the decision to reintroduce the shell and, critically, in what form.

Forces at play:

- Most agent CLIs expose only a stateless `!`-prefix passthrough where `cd`/env do not persist, output garbles, and interactive commands break the TUI. A real, persistent shell is genuinely more useful, but a *plain* real shell is only parity with Toad (Python/Textual, AGPL + commercial), which already ships one.
- Kitten's sole defensible angle is that shell state could become **shared context two agents hand back and forth** - something Toad (single-agent) and Warp (no portable hand-off) structurally cannot do.
- Technical risk collapsed in Dec 2025: Bun v1.3.5 shipped native PTY (`Bun.Terminal`), and `ghostty-opentui` provides an OpenTUI-native ANSI/VT renderer with color + scrollback. V1 is now a medium, de-risked integration, not a research project.
- A five-advisor council converged on a sharp constraint: a stateless passthrough **cannot** produce trustworthy `cwd`/env/exit-codes, so "capture on passthrough" would feed the hand-off lies and poison the very moat test it is meant to run. A real persistent PTY is a *prerequisite* for a clean test, not a detour. The council also held that a hand-off seam nobody consumes proves nothing, so the ride-along must be *in* V1 or the shell should stay deferred.

## Decision

Build V1 as a **real, persistent PTY shell that feeds a minimal, opt-in, human-curated hand-off ride-along**, scoped as follows:

1. **Real shell substrate.** Spawn one long-lived interactive `$SHELL` in a `Bun.Terminal` PTY, mirroring the existing `AgentConnection`/`TransportFactory` runtime pattern and reusing the ~16 ms output-coalescing frame scheduler. Persistent `cwd`/env, tab completion, history, and full-color output come from the real shell, not from bespoke code.
2. **Semantic shell-state slice.** Model shell state as a structured store slice - `cwd` plus a bounded ring of command records (command, captured output, exit code) - rendered from a bounded ring buffer, never a raw pixel grid. Quarantine the VT parser exactly as the ACP SDK is quarantined.
3. **Hand-off ride-along IN V1.** `cwd` and a bounded set of recent command records can be attached to the hand-off bundle. Attachment is **opt-in and curated by the human in the existing preview**, passes the existing secret redactor plus the mandatory preview gate, and **excludes environment variables** in V1.
4. **Generalized focus + interrupt.** Replace the agent-only `focusedAgentId` with a `focusedPaneId` over a discriminated pane union (`agent | shell`). Rebind `Ctrl+C` so that, when the shell is focused, it interrupts the shell's foreground child process rather than quitting the app.
5. **Graceful degradation for interactive apps.** Detect the alt-screen escape (`ESC[?1049h`, plus legacy `?47h`/`?1047h`) at the VT-parser boundary and decline full-screen interactive apps cleanly (clear message, forward SIGINT, reap the child) instead of half-rendering. In-pane rendering of full-screen TUIs (vim/htop/lazygit) is deferred to V2.

## Alternatives Considered

### Alternative 1: Plain persistent shell, hand-off "architected for later"

- **Description**: Ship the real shell with only a read-only seam exposing `cwd`/output; defer any hand-off wiring to V2.
- **Pros**: Smallest surface; clean separation; delivers "never leave the cockpit."
- **Cons**: A seam nobody consumes proves nothing; leaves Kitten at pure Toad parity with no differentiation shipped; does not test the moat, so it cannot justify reversing the prior deferral.
- **Why rejected**: The council showed the only defensible reason to build the shell is to feed the hand-off. Deferring that makes the whole feature a parity play the prior V1 already, correctly, declined.

### Alternative 2: Invert - thin capture on the existing `!` passthrough first

- **Description**: Skip the real shell; capture `cwd` + last-N output from the stateless passthrough, inject into the hand-off, measure the moat metric, then build the shell only if validated.
- **Pros**: Cheapest possible moat experiment; front-loads the differentiator; respects the prior deferral.
- **Cons**: A stateless passthrough has no durable `cwd`/env and no reliable command boundaries or exit codes, so it can only ship a *guess* on exactly the highest-value field (`cwd`). A null/negative result becomes uninterpretable - a poisoned experiment.
- **Why rejected**: The real shell is the cheapest source of *trustworthy* state; testing the moat on an untrustworthy substrate would produce a confident wrong conclusion. (Partially adopted: the class-1 fields - command/output/exit-code - and the bundle/redaction path can be de-risked early, and the ride-along is pulled into V1 so the moat test actually runs.)

### Alternative 3: Full interactive shell (in-pane vim/htop) in V1

- **Description**: Include full alt-screen VT emulation so interactive TUIs run inside the pane.
- **Pros**: Closes the "take-the-wheel" intervention gap; matches Toad's headline capability.
- **Cons**: Full-screen VT emulation, alt-screen/primary buffer modeling, and mouse handling are a much larger, research-adjacent build and the ecosystem's genuinely hard problem.
- **Why rejected**: Disproportionate to a first release; graceful degradation covers the honesty concern, and a full-window PTY takeover (open question) is a cheaper path to the intervention moment for V2.

## Consequences

### Positive

- Ships the one differentiated story no competitor can match: a persistent real shell whose state two agents hand back and forth.
- Trustworthy `cwd`/exit-codes make the moat test (does shell context reduce re-explanation / lift hand-off success) clean and interpretable.
- Reuses proven internal patterns (imperative runtime in the controller, pure reducer into an immutable store, frame-coalescing scheduler, arbitrary-text redactor), keeping the layered architecture coherent.

### Negative

- Larger than a plain shell: pulls the focus-model generalization and the hand-off ride-along into V1 rather than deferring them.
- Reverses a prior prioritization decision; the burden of proof on the moat metric is real and partly unmet (see Risks).

### Risks

- **Secret leakage into a second vendor's cloud** - shell output is the densest secret source flowing through a false-negative-biased redactor into agent B. *Mitigation:* opt-in + curated attachment, env excluded, redactor + mandatory preview, shell-specific redaction patterns, treat output as untrusted (prompt-injection into agent B).
- **`Ctrl+C`/focus landmine** - interrupt currently quits the app. *Mitigation:* fix focus generalization + `Ctrl+C` rebinding first, as a ship-blocker.
- **Intervention gap** - graceful refusal still forces a context switch for vim/lazygit/htop, the modal take-the-wheel moment. *Mitigation / open question:* evidence that alt-screen is a minority of interventions, or a full-window PTY takeover in V1.5/V2.
- **Measurability** - "reduced context-switching" cannot be cleanly proven with content-free, opt-in telemetry and no OS-level window-switch visibility. *Mitigation:* Kitten-mediated observable proxies (an in-cockpit "run externally" action), an optional randomized holdout among opted-in users, and honest self-report; do not overclaim.
- **Platform** - Bun PTY is POSIX-only; no Windows yet. *Mitigation:* document as a constraint.

## Implementation Notes

- Runtime: mirror `src/agent/transport.ts` `TransportFactory` + `AgentConnection`; own and dispose via `SessionController`; reuse `createFrameScheduler`.
- State: add a pure `shellReducer` + `shell` slice to `AppState`; memoized selectors; bounded ring buffer for scrollback and for the command ring.
- Render: quarantine `ghostty-opentui` (or an `@xterm/headless` model) behind one module; render styled runs into a `<scrollbox>` mirroring `ConversationView`.
- Hand-off: add an optional `shell` field to `HandoffBundle` (`src/core/types.ts`), populate + redact in `bundleAssembler`, compose a block in `composeHandoffBlocks`, and add a droppable "Shell context" section in `HandoffPreview`.
- Redaction reuse: `createSecretRedactor().redact()` already handles arbitrary text.
