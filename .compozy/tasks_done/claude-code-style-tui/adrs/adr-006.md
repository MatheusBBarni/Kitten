# ADR-006: Status Bar - Typed Slot Contract, Delegated Data Plumbing, and Honest Hand-off Affordance

## Status

Accepted

## Date

2026-07-09

## Context

The reskinned dual-agent status bar must show per-agent focus, run-state, model, and context-usage %, plus a shared git branch and cwd, and an always-visible hand-off affordance - inside the hard 80-column budget asserted by `src/ui/StatusStrip.test.tsx` (`expect([...strip].length).toBe(80)`).
Codebase exploration found that model/effort and context-usage arrive on ACP events Kitten currently drops (`config_option_update`, `usage_update`), and those two data features have their own pending task packets (`model-effort-selector`, `agent-usage-gauge`).
The hand-off is invisible today: `handoff.begin()` returns a bare boolean and silently no-ops when an overlay is open, the target is not ready, or the source has no turns; the shell ignores the return value.

## Decision

Specify the status bar's presentation and a typed slot contract, delegate the model/effort and context data plumbing, and make the hand-off affordance honest.

- **Slot contract.** The bar reads each signal through a narrow, curried selector that returns a typed value or `null` (hide-when-absent). This techspec defines the selector signatures (e.g. `selectSessionModel`, `selectSessionContext`) and the bar consumes them; they return `null` until their owning features land, so no layout reflow is needed later.
- **Delegated plumbing.** The ACP translate + reducer wiring for `config_option_update` (model/effort) and `usage_update` (context %) is owned by `model-effort-selector` and `agent-usage-gauge`; this techspec does not build it.
- **Priority-collapse budget.** The bar declares a fixed slot order with an explicit shed order as width tightens (branch, then context%, then effort), empty slots collapse to zero width, and an updated `StatusStrip.test` enforces the 80-column budget with both agents at their longest state.
- **Focus vs run-state.** These stay two orthogonal signals per agent, conveyed by glyph + label in addition to color.
- **Honest hand-off.** `handoff.begin()` changes to return a discriminated result `{ ok: true } | { ok: false; reason: "overlay-open" | "no-target" | "empty-source" }`. The bar's hand-off affordance derives its enabled/disabled state and reason directly from the store (target readiness via `controller.isReady`, source turn count, open-overlay), so the reason is visible before the key is pressed.

## Alternatives Considered

### Alternative 1: Full plumbing in this techspec

- **Description**: Translate `config_option_update` + `usage_update` into new `SessionState` fields, reducer, and selectors here, absorbing the two pending packets.
- **Pros**: One self-contained spec; context% present immediately.
- **Cons**: Duplicates/supersedes two existing efforts; enlarges scope well beyond chrome; risks divergent implementations.
- **Why rejected**: Violates ADR-002's additive phasing and the reskin's chrome-only scope.

### Alternative 2: Presentation only, no contract

- **Description**: Specify the visual restructure and git branch; leave even the slot contract for later.
- **Pros**: Leanest now.
- **Cons**: Phase 3 integration is under-specified and may force a second bar refactor when the data lands.
- **Why rejected**: The typed contract is cheap now and prevents a later reflow.

### Alternative 3: Keep `begin()` boolean, show a transient message

- **Description**: Leave the hand-off API as-is and show a toast when a press no-ops.
- **Pros**: Smaller change.
- **Cons**: Reactive only - the affordance still looks the same whether or not it will work; less honest than a continuously-derived disabled state.
- **Why rejected**: The PRD wants the reason visible, not a post-press surprise.

## Consequences

### Positive

- The bar ships honest and partial now, and the flagship signals snap into pre-built slots without a reflow.
- The hand-off - Kitten's signature action - becomes discoverable and self-explaining.
- The 80-column contract is preserved as an enforced budget with a declared collapse order.

### Negative

- The `begin()` signature change touches `handoff.ts` and its one call site in `CockpitApp.tsx` (and tests).
- Some bar slots render empty until their features land.

### Risks

- **80-column overflow** when many signals are present. Mitigation: fixed slots + priority-collapse + the updated width test as the gate.
- **Selector churn** repainting the wrong agent's chip. Mitigation: curried per-session selectors compared by value/identity, following the existing narrow-subscription pattern.

## Implementation Notes

- Define selector signatures returning `null` today; the bar treats `null` as hide.
- Change `HandoffFlow.begin` return type and update the `CockpitApp` key handler to pass the reason to the affordance's derived state (the affordance itself reads the store, so the press path mainly stops the silent no-op).
- Update `StatusStrip.test.tsx` to assert the new width budget with the richest both-agents state.
