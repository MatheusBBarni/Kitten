# ADR-001: V1 Scope for the Claude Code-Style TUI Reskin

## Status

Accepted

## Date

2026-07-09

## Context

Kitten is a two-agent terminal cockpit (Bun + TypeScript + OpenTUI + React) that runs Claude Code and Codex side by side over the Agent Client Protocol and hands a live coding task between them.
The owner asked to "refactor the TUI to look like the Claude Code one," refined during clarification to a **Kitten-branded chrome reskin**: the dual-agent layout, the hand-off flow, and the interactions stay unchanged; Kitten keeps its own mascot, accent, and wording.
Three candidate V1 pieces were identified: a welcome banner with a kitten mascot, a rich always-on bottom status bar, and a prompt + palette restyle.
The stated goal is both first-impression polish and always-on operational clarity.

Several constraints and forces shape the decision:

- **Data plumbing gap.** The status bar's marquee signals are not all available today. `cwd`, agent display names, live status, and focus are present now. Model + effort, git branch, and context-window usage % each need new plumbing. Two of the three already have accepted ADRs and pending task packets in sibling feature folders (`model-effort-selector`, `agent-usage-gauge`, the latter already targeting the exact `StatusStrip` chip); git branch needs a fresh, off-render-path shell-out.
- **80-column contract.** `StatusStrip.test` asserts the strip fits exactly 80 columns with both agents at their longest state. Adding branch + model + context% for two agents fights that budget.
- **Theme discipline.** All color lives in `theme.ts` (no hard-coded color in components) with test invariants, and there is an accepted `settings-modal` ADR-005 palette-registry seam (`ThemePreference` + keyed registry). The reskin must be legible in dark, light, and no-truecolor terminals.
- **Rendering limits.** OpenTUI supports rounded borders, titles, flexbox, ascii-font wordmarks, and character-cell sprites, but has no image protocol (Kitty/SIXEL/iTerm2). A mascot must be hand-authored character-cell art, which renders inconsistently across fonts and terminals.
- **Market lessons.** The genre convention (Claude Code, Gemini CLI, Copilot CLI, Charm Crush) is a banner plus a persistent status line. The single most-cited Claude Code complaint is an unskippable, slow splash. Animation is disproportionately expensive. Claude Code's own accent moved orange to blue, undocumented, so it is a moving imitation target.

A five-advisor council (pragmatic-engineer, architect-advisor, product-mind, devils-advocate, the-thinker) debated the scope and converged with preserved dissent.

## Decision

Ship all three pieces in V1, sequenced and constrained:

1. **Prompt + palette restyle.** Rounded borders, generous spacing, a chevron prompt, and Kitten's own warm accent registered as a preset on the ADR-005 palette registry. Pure presentation, reads every color through `theme.ts`. No new data. This is the honest, low-risk slice and lands first.
2. **Welcome banner + simple mascot.** Static, instant, idle-screen content (rendered in the conversation region's empty state), never a blocking or unskippable splash. The mascot is a small, deterministic ASCII / box-drawing kitten constrained to ANSI-safe colors and a fixed character grid, with a one-line greeting fallback under narrow width or limited terminal capability. Any elaborate or animated mascot is explicitly deferred.
3. **Restructured dual-agent status bar.** The signature surface. Renders the signals Kitten owns end-to-end today (focused agent, per-agent model when the session reports it, live status, cwd, and a cheap real git-branch read) and lights up model + effort and context-usage % additively via the existing `model-effort-selector` and `agent-usage-gauge` seams. Governed by two hard rules: **hide-when-absent** (an unavailable signal occupies zero width and is never rendered as a fake or placeholder number) and an explicit **80-column priority-collapse budget** (fixed slots with a declared shed order as width tightens), enforced by an updated `StatusStrip.test`.

The framing north star is "make Kitten's dual-agent cockpit feel calm, clear, and unmistakably Kitten," using Claude Code's genre grammar as a floor rather than a clone target.

## Alternatives Considered

### Alternative 1: Defer the status bar until fully plumbed

- **Description**: Ship only the palette/prompt restyle and banner now; hold the rich status bar until model + effort and context-usage land.
- **Pros**: Avoids any appearance of a half-built bar; no coupling to pending features.
- **Cons**: Forfeits a real, honest improvement over today's name + status one-liner; wastes the decoupling seams that already exist; delays the product's signature surface.
- **Why rejected**: The council resolved that a hide-when-absent slot contract degrades honestly (absence rendered as absence, never a fake number), so the bar can ship truthfully now and grow additively. Deferral was over-correction.

### Alternative 2: Near-exact Claude Code clone

- **Description**: Mirror Claude Code's chrome, accent, wording, and status format as closely as possible.
- **Pros**: Instant familiarity; zero learning cost.
- **Cons**: Produces "a strictly worse Claude Code with a cat on it"; chains Kitten's identity to a moving, undocumented target; abandons the dual-agent differentiator.
- **Why rejected**: The owner explicitly chose a Kitten-branded take, and the differentiated value is the two-agent hand-off, which a single-agent clone cannot express.

### Alternative 3: Big-bang paradigm redesign

- **Description**: Reshape the interaction model toward a single-column or tabbed stream and rebuild how the two agents and the hand-off are surfaced.
- **Pros**: Could unlock a fundamentally different UX.
- **Cons**: Touches core UX and the hand-off product; high risk; far beyond "reskin."
- **Why rejected**: Out of scope. The owner scoped this as chrome only, with layout, hand-off, and interactions unchanged.

## Consequences

### Positive

- Delivers the entire first-impression-polish half of the goal with a small, low-risk diff (palette/prompt/banner) that touches no data flows.
- Establishes the dual-agent status bar as Kitten's signature surface, with honest degradation and stable seams that later features light up without a second layout reflow.
- Reuses the ADR-005 palette registry and the two pending status-signal seams instead of duplicating them; adds only one thin git-branch read-model.
- Keeps the reskin unmistakably Kitten and avoids chasing Claude Code's undocumented visual drift.

### Negative

- The status bar's full clarity payoff is coupled to the `model-effort-selector` and `agent-usage-gauge` features landing; until then the bar is an honest but partial improvement.
- Hand-authored cell-art mascot carries a cross-terminal rendering tax even when simplified, and needs a capability-gated fallback.
- The 80-column budget forces a declared truncation order rather than showing every signal at once on narrow terminals.

### Risks

- **Sequencing risk**: if model + effort and context-usage are not genuinely next in the queue, the status bar reads as scaffolding rather than the clarity win. Mitigation: confirm those two packets are prioritized right after the reskin; ship the bar honest and additive regardless.
- **80-column collision**: cramming signals breaks the width contract. Mitigation: fixed slots, explicit priority-collapse (shed branch, then context%, then effort), empty slots collapse to zero width, updated test as the enforced budget.
- **Cross-terminal mascot fragility**: half-block art can render as garbage on some terminals, destroying the trust it was meant to build. Mitigation: ANSI-safe fixed-grid art, width and capability gating, one-line greeting fallback; defer elaborate/animated art.
- **Splash annoyance**: a banner shown every launch becomes noise. Mitigation: static, instant, non-blocking, quiet on relaunch.
- **Theme regressions**: violating the no-hard-coded-color rule or the palette test invariants. Mitigation: route all color through the palette registry preset; keep light/dark parity and no-truecolor fallback.

## Implementation Notes

- Register Kitten's warm accent as a named preset on the ADR-005 palette registry; extend `theme.ts` palette objects rather than inlining any hex in components.
- The welcome banner's natural home is the conversation region's empty state (today's single muted `EMPTY_TRANSCRIPT_HINT` line).
- Each status-bar chip subscribes narrowly to its own store slice (ADR-004) so one agent's update never repaints the other; the git-branch provider shells out off the render path and publishes into the external store.
- Any palette or preset picker overlay is an absolutely-positioned box, not a React Portal (ADR-004).
