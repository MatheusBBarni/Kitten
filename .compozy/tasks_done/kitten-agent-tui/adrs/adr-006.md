# ADR-006: Distribution as a Compiled Standalone Binary

## Status

Accepted

## Date

2026-07-07

## Context

A PRD success metric is time-to-first-agent-response under 60 seconds from a clean install, and the target users are terminal-native developers who expect a fast, prerequisite-free install (toad ships a one-line installer).
Kitten is a Bun/TypeScript app, and Bun can compile a project into a standalone executable.

## Decision

Kitten is distributed primarily as **per-platform standalone executables** produced by `bun build --compile` (darwin-arm64, darwin-x64, linux-x64, linux-arm64), delivered through a one-line curl installer.
It is also published to npm so users with Bun can run `bunx kitten`.
The compiled binary embeds the Bun runtime, so end users need no separate runtime.

## Alternatives Considered

### Alternative 1: npm / bunx package only

- **Description**: Publish to npm; users run `bunx kitten` or install globally.
- **Pros**: Simplest to ship; no cross-compilation.
- **Cons**: Requires the user to have Bun installed, adding a prerequisite and setup friction that works against the onboarding metric.
- **Why rejected**: The prerequisite undercuts the fast, clean first run.

### Alternative 2: Homebrew tap first

- **Description**: Distribute the binaries through a Homebrew tap plus curl.
- **Pros**: Familiar native install UX for mac/linux developers.
- **Cons**: Most release infrastructure to build and maintain for V1.
- **Why rejected**: Deferred; a curl installer plus npm covers V1, and a tap can be added later without changing the build.

## Consequences

### Positive

- No runtime prerequisite for end users; supports the under-60-second first run.
- One artifact per platform is simple to document and install.

### Negative

- Cross-compilation and release automation per platform must be set up.
- The binary is larger because it embeds the Bun runtime.

### Risks

- Compiled-binary edge cases (native FFI for OpenTUI's Zig core across platforms) - mitigate by validating each target artifact in CI on a real terminal before release.

## Implementation Notes

- Use `Bun.build` with `compile.target` per platform; keep the curl installer script minimal and checksummed.
- Publish the npm package as a secondary path so contributors and Bun users can run from source or `bunx`.
