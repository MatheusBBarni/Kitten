# ADR-003: MCP Server Domain Model and ACP Translation Boundary

## Status

Accepted

## Date

2026-07-09

## Context

ADR-001 decided that Kitten provisions MCP servers by injecting them into each agent's ACP session rather than writing agent config files.
This ADR fixes how that injection is wired through the codebase.

Three existing facts constrain the design:
- The layered-architecture rule (kitten-agent-tui ADR-003) forbids ACP SDK wire types from leaking past `src/agent`; the domain core never imports the SDK.
- `AppConfig` carries `providers: Record<ProviderKind, ProviderRecipe>`, `sessions`, and `telemetryEnabled`; there is no MCP field today.
- `AgentConnection.newSession(cwd: string)` takes only a cwd, and its implementation calls `connection.newSession({ cwd, mcpServers: [] })` with a hardcoded empty array.
- The ACP `McpServer` stdio shape uses `env: Array<EnvVariable>` (name/value pairs) and expects an absolute `command`, whereas Kitten config uses an `env` map and a bare command.

## Decision

Introduce an SDK-free domain type for an MCP server in `src/core/types.ts`, carrying the stdio fields Kitten supports in V1: a name, a command, arguments, and an env map.

Add a top-level `mcpServers` field to the config: a name-keyed map in the config file (validated by the strict user-config schema), normalized by the loader into a domain list on `AppConfig` with the name lifted from each key.
The list is global (one set for all agents) per ADR-002.

Widen `AgentConnection.newSession` to accept the resolved MCP server list alongside the cwd, and thread that list from the controller's per-session start path (which already holds `AppConfig`) down to the call site that currently passes `[]`.

Confine SDK translation to `src/agent/acpTranslate.ts`: a helper maps the domain MCP list to the SDK `McpServer` union, converting the env map to the name/value array shape and resolving the command to an absolute path.
No SDK MCP type appears outside `src/agent`.

## Alternatives Considered

### Alternative 1: Translate inline inside agentConnection

- **Description**: Build the SDK `McpServer[]` directly inside `agentConnection.newSession` rather than in `acpTranslate.ts`.
- **Pros**: Fewer files touched.
- **Cons**: Spreads SDK-shape construction across the adapter; `acpTranslate.ts` is the established home for domain-to-SDK mapping.
- **Why rejected**: Keeping all wire translation in `acpTranslate.ts` preserves the single-purpose boundary and keeps `agentConnection` about connection lifecycle.

### Alternative 2: Store MCP servers per provider on `ProviderRecipe`

- **Description**: Attach `mcpServers` to each provider recipe instead of a top-level global field.
- **Pros**: Would make per-agent lists trivial later.
- **Cons**: Contradicts ADR-002's single global list; duplicates the same list across providers in V1.
- **Why rejected**: V1 is one global list; a per-agent exclude (V2) is modeled as an overridable filter over the global list, not as separate per-provider lists.

## Consequences

### Positive

- The domain core stays SDK-free; all wire quirks (env-as-array, absolute command) are absorbed in one translator.
- A single global list matches the "declare once" product goal and the ADR-002 scope.
- The change is small and routes through the seam the protocol already exposes.

### Negative

- Widening `newSession` ripples to every `AgentConnection` implementation, including the `StubConnection` test fake, the `createOfflineConnection` fake in `selfCheck.ts`, and any in-memory ACP stub.
- A second config normalization path (map to list) is added to the loader.

### Risks

- **Signature drift across fakes.** Mitigation: the `AgentConnection` interface is the single contract; updating it surfaces every fake at compile time (TypeScript), so none is missed silently.
- **Command resolution failure.** A bare command that cannot be resolved to an absolute path must not crash boot. Mitigation: resolution lives in the translator with a skip-and-warn path (see ADR-004 for the failure stance).

## Implementation Notes

- Domain type and `AppConfig.mcpServers` live in `src/core/types.ts`; schema and map-to-list normalization in `src/config/configLoader.ts`.
- Translation helper in `src/agent/acpTranslate.ts`; `newSession` widening in `src/agent/agentConnection.ts`; threading in `src/app/controller.ts`.
- V1 schema accepts stdio entries only; a remote (http/sse) entry is a `ConfigError` at load.
